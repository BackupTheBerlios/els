Description

I am using the following terms:

  • Planning: I am planning to create such a module. There might be anything in the CVS, but it will normally not usable, not even for developers.
  • Development: the module is in development. There is something in the CVS and it should be usable, but it may contain bugs, have not all features or eat your hard disk. But reporting should be done via the develepers mailing list.
  • Alpha: Something is downloadable. The code may still contain bugs or eat your hard disk, but it should give developers (or other brave people) an idea on the general direction of the modules. Therefore it might be the case that not all planned features have been implemented. Bug reporting should be done via the develepers mailing list. Usually the CVS version is more advanced then the downloadable alpha.
  • Beta: Something is downloadable. It is even assumed to be bug less. However, it might still eat your hard disk. Hey, EVERY software can eat your hard disk ... that should not hinder you from installing this software. The CVS version might or might not be more advanced.
  • Released: once a module has been blessed to be "Released" it should blink and shine with all if it's bits. Go, install it immediately. Your reward will be eternally ... And, by the way: it may still eat your hard disk. All of this software comes free of charge and without any guarantee :-)
